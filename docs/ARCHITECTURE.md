# Архитектура Gimnasy Engine

## Слои

```
┌─────────────────────────────────────────────────────────────┐
│  Gimnasy.Editor      backend редактора (CLI): сцены, типы,    │
│                      иконки, компиляция шейдер-графов          │
├─────────────────────────────────────────────────────────────┤
│  Gimnasy.Runtime     игровой цикл, IRenderingServer, плеер     │
│  Gimnasy.Scripting   GameScript, загрузка сборок со скриптами  │
├─────────────────────────────────────────────────────────────┤
│  Gimnasy.Nodes       150+ типов узлов (2D/3D/UI/VFX/террейн…) │
├─────────────────────────────────────────────────────────────┤
│  Gimnasy.Core        математика, объекты/сигналы, дерево сцены,│
│                      сериализация, ресурсы, террейн, геометрия,│
│                      нодовый шейдер-граф                        │
└─────────────────────────────────────────────────────────────┘
```

`Gimnasy.Core` не имеет внешних зависимостей (только BCL) — собирается одним
лишь .NET SDK и полностью покрывается модульными проверками логики.

## Ключевые системы

### Дерево сцены
`Node` — базовый элемент. Узлы образуют дерево; `SceneTree` владеет корнем и
каждый кадр обходит дерево, доставляя колбэки `_Process` (раз в кадр) и
`_PhysicsProcess` (фиксированный шаг). Жизненный цикл:
`_EnterTree → _Ready → _Process*/_PhysicsProcess* → _ExitTree`.

### Система типов (ClassDB)
Классы, помеченные `[RegisteredType("Имя", "Категория")]`, находятся
рефлексией и регистрируются в `ClassDb`. Свойства с `[Export]` участвуют в
сериализации и в инспекторе. Это даёт единый источник правды для:
сериализации сцен, диалога «Add Node», скриптинга.

### Сигналы
Строковые сигналы (`Connect` / `EmitSignal` / `Disconnect`) на базовом классе
`GObject`. Снимок списка слушателей берётся на момент emission, поэтому
обработчики могут безопасно подключаться/отключаться во время вызова.

### Сериализация
`SceneSerializer`, `ResourceLoader`, `ShaderGraphIO` пишут JSON-подобный текст
через `JsonLike` (детерминированный вывод, чтение допускает комментарии и
висячие запятые). Значения конвертируются `VariantConverter`; векторы и цвета
сохраняются компактными массивами.

### Рендеринг
`IRenderingServer` — backend-агностичный контракт. `SceneRenderer` обходит
дерево и выдаёт draw-команды. В комплекте `NullRenderingServer` (headless);
GPU-бэкенд реализует тот же контракт (в планах).

### Скриптинг
Скрипт — это C#-подкласс узла с `[Export]`-полями, скомпилированный в сборку
игры. `ScriptHost` загружает сборку и регистрирует её типы, после чего сцены
могут на них ссылаться. (Hot-reload через Roslyn — в планах.)

## Принципы
1. **Данные отделены от рендера.** Узлы и ресурсы — это данные; бэкенд их
   интерпретирует. Поэтому движок работает и без GPU.
2. **Рефлексия вместо ручной регистрации.** Новый тип = новый класс с
   атрибутом; ничего больше править не нужно.
3. **Текстовые, diff-дружелюбные форматы.** Всё человекочитаемо и мержится.
